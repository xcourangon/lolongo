Lolongo

Lolongo is a small framework that provides a simple but efficient computational model:

  • A Context is a Data container.
  • A Data is uniquely identified by a Reference within a Context.
  • Data are not mutable and cannot be overwritten.
  • Contexts are structured as a hierarchical tree of Context.
  • A Function is evaluated within a Context.
  • A Function gets its inputs in a Context and puts its results in the same Context.
  • A Processor manages the evaluation of a set of Function in some Contexts.

Why 'Lolongo' ?

The word 'lolongo' stands for 'sequence' in Swahili that is:

a set of related events, movements, or things that follow each other in a particular order.

Tech

Lolongo is written in Java (1.5 or higher) and needs a jsr331 implementation (why not Choco ?).
